# Contributing

Thanks for your interest in contributing to this project! Below are some guidelines to help you make a high-quality contribution.

## How to Contribute

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/YourFeature`
3. Implement changes and add tests
4. Run tests locally: `pytest`
5. Commit and push: `git commit -am "Add feature"`
6. Open a pull request and describe your changes

## Coding Style
- Follow **PEP 8** style guidelines
- Use meaningful variable and function names
- Keep functions small and focused

## Tests
- Add tests for new features under `tests/`
- Ensure all tests pass before submitting a PR

## Reporting Bugs
- Open an issue with a clear description and steps to reproduce
- Include logs and minimal reproducible code when possible
